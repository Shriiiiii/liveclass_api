part 4: GraphQL Flow (POST /graphql)
===
For all requests below, set the HTTP method to POST and the URL to:
http://localhost:3000/graphql

4.1 Public Mutations (No token needed)
---
Signup a new user
Body (GraphQL):
mutation {
  signup(username: "alex", password: "password123", role: "student") {
    message
  }
}

Login to get token
Body (GraphQL):
mutation {
  login(username: "alex", password: "password123") {
    token
    username
  }
}
*Copy token from response and add to Headers: Authorization = Bearer <token>*

4.2 Student Flow (Authenticated - token needed in Headers)
---
Browse all classes
Body (GraphQL):
query {
  liveClasses {
    id
    subject
    trainer
    capacity
    status
  }
}

Show class details
Body (GraphQL):
query {
  liveClass(id: 27) {
    id
    subject
    trainer
    capacity
    status
    users {
      username
    }
  }
}

Enroll in a class
Body (GraphQL):
mutation {
  enrollInClass(liveClassId: 27) {
    message
  }
}

View my schedule
Body (GraphQL):
query {
  myClasses {
    id
    subject
    trainer
    status
  }
}

Student Logout
Body (GraphQL):
mutation {
  logout {
    message
  }
}

4.3 Trainer/Admin Flow (Authenticated - token needed in Headers)
---
Create a new class
Body (GraphQL):
mutation {
  createLiveClass(subject: "Intro to Rails", trainer: "manoj", capacity: 10, status: "Scheduled") {
    liveClass {
      id
      subject
    }
  }
}

Update class status
Body (GraphQL):
mutation {
  updateLiveClass(id: 27, status: "Active") {
    liveClass {
      id
      status
    }
  }
}

Delete a class
Body (GraphQL):
mutation {
  deleteLiveClass(id: 30) {
    id
  }
}
